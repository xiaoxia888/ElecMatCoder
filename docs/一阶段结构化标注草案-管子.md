# 一阶段结构化标注草案-管子

## 1. 目的

本文件只定义一件事：

- 管子模型的一阶段结构化标注方式

这里的前提已经固定：

- 不同分类使用不同模型
- 一个模型一套结构

因此本文件只讨论：

- 管子模型应该输出什么结构
- 管子模型下 `BODY` 如何归一
- 管子模型下 `TYPE` 哪些字段需要标注

---

## 2. 基本原则

### 2.1 一个模型一套结构

既然已经决定按分类拆模型，那么不同模型不必强行共用同一套 `TYPE` 结构。

也就是说：

- 管子模型只保留管子真正需要的字段
- 不引入法兰模型、弯头模型才需要的字段

### 2.2 `BODY` 按编码种类归一

`BODY` 不是按中文大类归一，而是按**最终编码种类**归一。

原则：

- 编码相同，可以归一
- 编码不同，必须分开

因此不能把所有管子相关描述都压成：

- `直管`

如果以下种类编码不同，则必须保留不同 `BODY`：

- `直管`
- `法兰管`
- `夹套钢管`

### 2.3 `TYPE` 字段标注分工

管子模型下：

- `BODY`：标注管子类编码种类
- `MANU`：标注制造方式
- `CONN`：标注连接方式
- `ENDS`：标注端部形式

例如：

- `无缝钢管`
  - 应标成 `BODY=直管`, `MANU=[SMLS]`

- `焊接钢管`
  - 应标成 `BODY=直管`, `MANU=[WELDED]`

- `锻制直管`
  - 应标成 `BODY=直管`, `MANU=[FORGED]`

前提是它们最终编码同属“直管”。

---

## 3. 管子模型的输出结构

当前建议管子模型的一阶段输出中，`TYPE` 只保留：

```json
"TYPE": {
  "BODY": "",
  "MANU": [],
  "CONN": [],
  "ENDS": []
}
```

说明：

- `BODY`：管子类编码种类
- `MANU`：制造方式
- `CONN`：连接方式
- `ENDS`：端部形式

---

## 4. 管子模型下 `BODY` 当前建议

当前至少保留以下标准 `BODY`：

- `直管`
- `法兰管`
- `夹套钢管`

后续若发现还有其他会直接影响种类编码的管子子类，再继续补充。

### 4.1 `直管`

适用于：

- `PIPE`
- `Pipe`
- `钢管`
- `管子`
- `TUBE`
- `无缝钢管`
- `焊接钢管`
- `SEAMLESS PIPE`
- `WELDED PIPE`
- 其他最终编码为普通直管的表达

### 4.2 `法兰管`

适用于：

- `FLANGED PIPE`
- `法兰管`
- `Flanged Pipe`
- `带法兰直管`
- 其他最终编码为法兰管的表达

### 4.3 `夹套钢管`

适用于：

- `夹套钢管`
- `夹套管`
- `JACKETED PIPE`
- `Jacketed Pipe`
- 其他最终编码为夹套钢管的表达

### 4.4 `BODY` 完整归一化映射

以下为当前数据集中已出现写法到标准 `BODY` 的归一化结果。

#### 归一到 `直管`
- `PIPE` -> `直管`
- `钢管` -> `直管`
- `无缝钢管` -> `直管`
- `WELDED PIPE` -> `直管`

#### 归一到 `法兰管`
- `FLANGED PIPE` -> `法兰管`
- `带法兰直管` -> `法兰管`

#### 归一到 `夹套钢管`
- `夹套钢管` -> `夹套钢管`
- `JACKETED PIPE` -> `夹套钢管`

---

## 5. 管子模型下其他 `TYPE` 字段

### 5.1 `MANU`

以下制造方式进入 `MANU`：

- `SMLS`
- `WELDED`
- `ERW`
- `EFW`
- `SAWL`
- `SAWH`
- `FORGED`

注意：

- 原文出现 `SEAMLESS` 时，统一标为 `SMLS`
- 原文出现 `无缝` 时，如需要标制造方式，也统一标为 `SMLS`

### 5.2 `CONN`

以下连接方式进入 `CONN`：

- `SW`
- `SF`
- `NPT`
- `THD`

### 5.3 `ENDS`

以下端部形式进入 `ENDS`：

- `FNPT`
- `MNPT`
- `SO`
- `WN`
- `MTE`
- `TSE`
- `FTE`
- `NPTF`

## 6. 示例

### 示例 1：普通直管

原文：

```text
PIPE / SMLS / ASME B36.19M Sch10S / A312 TP304 / BE / POLISHING DN20
```

推荐：

```json
"TYPE": {
  "BODY": "直管",
  "MANU": ["SMLS"],
  "CONN": [],
  "ENDS": []
}
```

## 7. 当前结论

管子模型当前采用以下规则：

- `TYPE` 只保留：
  - `BODY`
  - `MANU`
  - `CONN`
  - `ENDS`
- `BODY` 按最终编码种类区分
- 当前至少区分：
  - `直管`
  - `法兰管`
  - `夹套钢管`

后续待补充：

- `PIPE NIPPLE` 是否仍归管子模型，还是转入其他管件模型
- `短管`、`短节` 是否继续归到 `直管`
- `金属软管` 是否继续并入管子模型，还是单列
