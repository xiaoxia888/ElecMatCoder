# 一阶段结构化标注草案-法兰

## 1. 目的

本文件只定义一件事：

- 法兰模型的一阶段结构化标注方式

这里的前提已经固定：

- 不同分类使用不同模型
- 一个模型一套结构

因此本文件只讨论：

- 法兰模型应该输出什么结构
- 法兰模型下 `BODY` 如何归一
- 法兰模型下 `TYPE` 哪些字段需要标注

---

## 2. 基本原则

### 2.1 一个模型一套结构

既然已经决定按分类拆模型，那么法兰模型只保留法兰真正需要的字段。

### 2.2 `BODY` 按编码种类归一

`BODY` 不是按中文大类归一，而是按**最终编码种类**归一。

原则：

- 编码相同，可以归一
- 编码不同，必须分开

因此法兰模型下，至少需要区分：

- `法兰`
- `螺纹法兰`
- `承插焊法兰`
- `带颈对焊法兰`
- `带颈平焊法兰`
- `板式平焊法兰`
- `对焊法兰`
- `法兰盖`
- `盲法兰`
- `8字盲板`
- `夹套法兰`
- `松套法兰`
- `插板`

### 2.3 `TYPE` 字段标注分工

法兰模型下：

- `BODY`：标注法兰类编码种类
- `CONN`：标注连接方式
- `SEAL`：标注密封面形式
- `ENDS`：标注端口型值

例如：

- `THREADED FLANGE`
  - 应标成 `BODY=螺纹法兰`, `CONN=[THD]`

- `SOCKET WELD FLANGE`
  - 应标成 `BODY=承插焊法兰`, `CONN=[SW]`

- `BLIND FLANGE`
  - 应标成 `BODY=盲法兰`

- `SO FLANGE`
  - 应标成 `BODY=带颈平焊法兰`

- `PADDLE BLIND`
  - 应标成 `BODY=8字盲板`

---

## 3. 法兰模型的输出结构

当前建议法兰模型的一阶段输出中，`TYPE` 只保留：

```json
"TYPE": {
  "BODY": "",
  "CONN": [],
  "SEAL": [],
  "ENDS": []
}
```

说明：

- `BODY`：法兰类编码种类
- `CONN`：连接方式
- `SEAL`：密封面形式
- `ENDS`：端口型值

## 7. 当前结论

法兰模型当前采用以下规则：

- `TYPE` 只保留：
  - `BODY`
  - `CONN`
  - `SEAL`
  - `ENDS`
- `BODY` 按最终编码种类区分
- `SO`、`WN` 这类直接决定法兰子类型的代码，进入 `BODY`，不进入 `ENDS`
- `RF` 作为标准密封面进入 `SEAL`
- 当前至少区分：
  - `法兰`
  - `带颈对焊法兰`
  - `带颈平焊法兰`
  - `板式平焊法兰`
  - `对焊法兰`
  - `法兰盖`
  - `盲法兰`
  - `8字盲板`
  - `夹套法兰`
  - `松套法兰`
  - `插板`

- `法兰盖` 与 `盲法兰` 的边界例子
