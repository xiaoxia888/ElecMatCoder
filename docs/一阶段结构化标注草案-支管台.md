# 一阶段结构化标注草案-支管台

## 1. 目的

本文件只定义一件事：

- 支管台模型的一阶段结构化标注方式

这里的前提已经固定：

- 不同分类使用不同模型
- 一个模型一套结构

因此本文件只讨论：

- 支管台模型应该输出什么结构
- 支管台模型下 `BODY` 如何归一
- 支管台模型下 `TYPE` 哪些字段需要标注

---

## 2. 基本原则

### 2.1 一个模型一套结构

既然已经决定按分类拆模型，那么支管台模型只保留支管台真正需要的字段。

### 2.2 `BODY` 按编码种类归一

`BODY` 按**最终编码种类**归一。

原则：

- 编码相同，可以归一
- 编码不同，必须分开

当前支管台模型下，至少需要区分：

- `支管台`
- `承插焊支管台`
- `螺纹支管台`
- `弯头支管台`
- `斜支管台`
- `短节支管台`
- `管接头`
- `加强管接头`
- `短管接头`
- `异径管接头`
- `半管接头`

### 2.3 `TYPE` 字段标注分工

支管台模型下：

- `BODY`：标注支管台类编码种类
- `MANU`：标注制造方式
- `CONN`：标注连接方式
- `ENDS`：标注端部形式

例如：

- `WELDOLET`
  - 应标成 `BODY=支管台`

- `SOCKOLET`
  - 应标成 `BODY=承插焊支管台`, `CONN=[SW]`

- `THREDOLET`
  - 应标成 `BODY=螺纹支管台`, `CONN=[THD]`

- `ELBOLET`
  - 应标成 `BODY=弯头支管台`

- `LATROLET`
  - 应标成 `BODY=斜支管台`

- `NIPOLET`
  - 应标成 `BODY=短节支管台`

- `管接头`
  - 应标成 `BODY=管接头`

- `加强管接头`
  - 应标成 `BODY=加强管接头`

- `短管接头`
  - 应标成 `BODY=短管接头`

- `异径管接头`
  - 应标成 `BODY=异径管接头`

- `半管接头`
  - 应标成 `BODY=半管接头`

---

## 3. 支管台模型的输出结构

当前建议支管台模型的一阶段输出中，`TYPE` 保留：

```json
"TYPE": {
  "BODY": "",
  "MANU": [],
  "CONN": [],
  "ENDS": []
}
```

说明：

- `BODY`：支管台类编码种类
- `MANU`：制造方式
- `CONN`：连接方式
- `ENDS`：端部形式

## 5. 支管台模型下其他 `TYPE` 字段

### 5.1 `MANU`

以下制造方式进入 `MANU`：

- `SMLS`
- `WELDED`
- `ERW`
- `EFW`
- `SAWL`
- `SAWH`
- `FORGED`

补充规则：

- 原文出现 `SEAMLESS` 时，统一标为 `SMLS`
- 原文出现 `无缝` 时，如需要标制造方式，也统一标为 `SMLS`

### 5.2 `CONN`

以下连接方式进入 `CONN`：

- `SW`
- `SF`
- `NPT`
- `THD`

### 5.3 `ENDS`

以下端部形式进入 `ENDS`：

- `SO`
- `FNPT`
- `MNPT`
- `MTE`
- `TSE`
- `FTE`
- `NPTF`

## 7. 当前结论

支管台模型当前采用以下规则：

- `TYPE` 保留：
  - `BODY`
  - `MANU`
  - `CONN`
  - `ENDS`
- `BODY` 当前至少区分：
  - `支管台`
  - `弯头支管台`
  - `斜支管台`
  - `短节支管台`
  - `管接头`
  - `加强管接头`
  - `短管接头`
  - `异径管接头`
  - `半管接头`
- `管接台 / 支管台 / 支管座`
  - 当前统一归到 `支管台`
- `WELDOLET / SOCKOLET / THREDOLET`
  - 当前统一归到 `支管台`
  - 连接方式通过 `CONN` 表达
- `管接头 / 加强管接头 / 短管接头 / 异径管接头`
  - 当前分别单独保留
- `半管接头`
  - 在支管台模型内单独保留
- `HALF COUPLING / HALF CPLG`
  - 当前统一归到 `半管接头`

后续待补充：

- `OLET` 是否永远归到 `支管台` 的边界例子
- `短管接头` 当前数据样本补充后，再补完整归一化映射
- `半管接头` 与普通 `支管台` 的边界例子
