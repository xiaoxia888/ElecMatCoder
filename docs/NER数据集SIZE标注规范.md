# NER 数据集 SIZE 标注规范

## 1. 提取什么

从原文中提取所有与尺寸有关、能用于后续编码的尺寸信息，按类别拆到 `SIZE` 对象中：

- `DN`：工程直径 / 公称直径
- `OD`：外径
- `INCH`：英寸尺寸

格式：

```json
"SIZE": {
  "DN": [],
  "OD": [],
  "INCH": []
}
```

空数组不输出。

## 2. 值格式怎么统一

1. `DN` 统一写成 `DN+数字`
2. `OD` 统一只保留数字本体，不保留 `mm`、`φ`、`Φ`、`Ø`、`OD`
3. `INCH` 统一写成 `数字+"`

## 3. 归类规则

### `DN`

以下都统一成 `DN+数字`：

- `20`
- `D20`
- `DN20`

前提：该值在当前样本里表示的是工程直径 / 公称直径。

### `OD`

以下都统一成纯数字：

- `57`
- `57mm`
- `φ57`
- `Φ57`
- `Ø57`
- `OD57`
- `外径57mm`

如果原文是外径组，整体都归 `OD`，例如：

- `Φ273.0×219.1/5.00×4.00` -> `OD = ["273.0", "219.1"]`

### `INCH`

以下都统一成 `数字+"`：

- `2"`
- `NPS2`
- `6x2"` -> `INCH = ["6\"", "2\""]`

## 4. 多值怎么存

1. 同一类别出现多个值时，按原文顺序放入数组
2. 异径件保留全部尺寸，不丢值
3. 重复值去重后保留一次

## 5. 判定优先级

1. 能明确判断为 `DN` 的，归 `DN`
2. 能明确判断为 `OD` 的，归 `OD`
3. 能明确判断为英寸的，归 `INCH`
4. 无法稳定判断时，进入人工复核，不强行猜测

## 6. 注意事项

1. `SIZE` 追求的是编码友好的统一输出，不是原样抄写
2. `THICKNESS` 不放进 `SIZE`
3. `DN` 与 `OD` / `INCH` 可以同时保留，只要它们表达的是不同维度
