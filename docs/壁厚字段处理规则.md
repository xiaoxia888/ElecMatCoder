# 壁厚字段处理规则

本文档用于说明 `THICKNESS` 字段在训练数据、编码映射和 `ThicknessProcessor` 中的处理口径。

## 目标

- 统一壁厚常见写法，减少标签稀疏
- 在不改变语义的前提下，把同义表达收敛到稳定编码
- 对少量 OCR/粘连/脏样本做可控归一

## 统一输出格式

- `SCH20` -> `S20`
- `SCH40S` -> `S40S`
- `S-40` -> `S40`
- `40S` -> `S40S`
- `8.1mm` -> `8.1MM`
- 多段壁厚统一用 `X` 连接
- 相邻重复段去重

示例：

- `SCH20(L)*SCH40(S)` -> `S20XS40`
- `S-40S, S-80S` -> `S40SXS80S`
- `THK=6.3mmX6.3mm` -> `6.3MM`

## 当前表达类型

基于 `data/pipe/llm_lora/encoding_mappings.json` 中 `THICKNESS` 映射统计，当前主要表达方式如下：

- `SCH` 体系：如 `SCH20`、`SCH40S`、`SCHSTD`
- 双值/多值表达：如 `SCH20*SCH40`、`SCH20/SCH40`、`4mm/16.6mm`
- 大小端标记：如 `SCH40 L x SCH80 S`、`SCH20(L)×SCH40(S)`
- `S-` 体系：如 `S-20`、`S-40S`、`S-STD`
- `40S/80S` 简写：如 `40S`、`10Sx40S`
- 数值厚度：如 `8.1mm`、`THK=6.3mm`、`T=3.0mm`
- 罗马数字前缀样式：如 `II-3/2.5`、`Ⅱ-4×3`
- 下划线/OCR 脏样本：如 `Sch20_S`、`Sch40_STD`、`STD_STD`
- 无分隔符粘连：如 `SCH20SCH40`、`S40S20`、`S40SS20S`

## 处理规则

### 1. `SCH` 体系

- `SCH20` -> `S20`
- `SCH20S` -> `S20S`
- `SCHSTD` -> `STD`
- `SCHXS` -> `XS`
- `SCHXXS` -> `XXS`

### 2. `S-` 体系

- `S-20` -> `S20`
- `S-10S` -> `S10S`
- `S-STD` -> `STD`
- `S-XS` -> `XS`
- `S-XXS` -> `XXS`

### 3. `40S` 简写

- `40S` -> `S40S`
- `80S` -> `S80S`
- `10Sx40S` -> `S10SXS40S`

说明：

- 该类表达默认视为数字加 `S` 后缀的壁厚写法，不视为尺寸。

### 4. `mm / THK / T=` 数值体系

- `8.1mm` -> `8.1MM`
- `T=3.0mm` -> `3MM`
- `THK=6.0X4.5mm` -> `6MMX4.5MM`
- 纯数字默认按毫米处理：`6.3` -> `6.3MM`

### 5. 多段分隔符统一

以下分隔符统一视为多段连接：

- `x`
- `X`
- `×`
- `*`
- `/`
- `,`

示例：

- `SCH20*SCH40` -> `S20XS40`
- `SCH20/SCH40` -> `S20XS40`
- `4mm/16.6mm` -> `4MMX16.6MM`

### 6. 大小端标记处理

下列大小端标记只表示位置，不属于壁厚本体：

- `(L)`
- `(S)`
- 裸 `L`
- 裸 `S`

示例：

- `SCH40 L x SCH80 S` -> `S40XS80`
- `SCH20(L)×SCH40(S)` -> `S20XS40`

注意：

- 裸 `L/S` 必须在删空格前去掉，否则会粘成 `SCH40LxSCH80S`，无法再正确拆分。

### 7. 无分隔符粘连拆分

当前处理器已支持无显式分隔符的连续 token 回溯拆分。

示例：

- `SCH20SCH40` -> `S20XS40`
- `SCH20SCH40S` -> `S20XS40S`
- `S40S20` -> `S40XS20`
- `40S20S` -> `S40SXS20S`
- `S40SS20S` -> `S40SXS20S`
- `S40S40S` -> `S40S`

### 8. 罗马数字前缀样式

`II-` / `Ⅱ-` 在当前数据中表示系列写法，不保留到最终壁厚编码中。

示例：

- `II-3` -> `3MM`
- `Ⅱ-4×3` -> `4MMX3MM`
- `II-2.5/2.5` -> `2.5MM`

说明：

- 处理策略是在预处理阶段移除 `II-` / `Ⅱ-` 前缀，再走普通数值厚度逻辑。

## 下划线脏样本别名

下列样本来自现有映射表，语义不完全一致，因此在处理器中按显式别名归一：

- `Sch20_S` -> `S20S`
- `Sch40S_S` -> `S40SXS`
- `Sch40_S` -> `S40XS`
- `Sch40_STD` -> `S40XSTD`
- `STD_STD` -> `STD`
- `STD_Sch2` -> `STDXS2`
- `Sch20_STD` -> `STDXS20`
- `XS_XS` -> `XS`

说明：

- 这类下划线样本不建议继续扩展为一条宽泛规则。
- 更安全的做法是：把它们视为映射表驱动的脏样本别名。

## 建议的数据标注口径

- 优先标注清晰、语义稳定的原始壁厚值
- 同语义的常见格式可以统一：
  - `SCH 40`、`SCH40`、`Sch.40` 允许统一到同一输出
  - `S-40`、`SCH40` 不建议在标注阶段互相改写，但编码阶段应收敛
- 对明显 OCR/脏样本，建议保留原文并在编码规则层处理
- 对极少量孤例脏样本，不要为了 1 条样本发明大而泛的规则

## 当前处理器状态

- `60` 通过
- `0` 失败

后续如果新增壁厚写法，建议同步做两件事：

1. 往 `encoding_mappings.json` 增加映射样本
2. 往 `ThicknessProcessor` 的自测列表补对应 case
