# 材质标注规则

本文档仅用于说明一阶段 `MATERIAL` 字段的标注口径。

适用数据集：

- `data/pipe/llm_lora/ner_data_new_schema.json`

## 1. 标注目标

`MATERIAL` 不再使用单字符串，而采用结构化对象标注，用于表达：

- 材质关系
- 执行标准
- 材质牌号表达
- 特殊要求

## 2. 统一结构

```json
"MATERIAL": {
  "RELATION": "single",
  "ITEMS": [
    {
      "EXEC_STANDARD": "ASTM A403",
      "MATERIAL_GRADE_CODE": "WP316L",
      "SPECIAL_REQ": []
    }
  ]
}
```

字段说明：

- `RELATION`：材质关系，当前只使用 `single` 或 `alternative`
- `ITEMS`：材质项列表
- `EXEC_STANDARD`：材质执行标准主体，如 `ASTM A403`、`A182`、`A53`、`A106`、`API5L`
- `MATERIAL_GRADE_CODE`：完整材质牌号表达，按原文表面形式保留
- `SPECIAL_REQ`：特殊要求列表，按原文表面形式保留

## 3. 总原则

### 3.1 原文优先

- 以原文中的材质表达为准
- 优先保留原文表面形式
- 不在一阶段提前做编码归一
- 不凭经验补充原文中不存在的信息

### 3.2 结构化但不过度改写

允许：

- 拆出明确的材质标准号
- 拆出明确的特殊要求
- 处理明显的标准号前后倒置

不允许：

- 将原文 `GRADE B` 强行改成 `GR.B`
- 将原文 `GALVANIZED` 强行改成 `镀锌`
- 将原文 `CE` 直接改成 `抗硫`
- 将原文中不存在的标准前缀补全到字段里

### 3.3 空字段不输出

- `EXEC_STANDARD` 为空时不写该键
- 不写空字符串占位

## 4. 各字段规则

### 4.1 `RELATION`

当前使用规则：

- 单一材质：`single`
- 二选一 / 可替代材质：`alternative`

示例：

```json
"MATERIAL": {
  "RELATION": "alternative",
  "ITEMS": [
    {
      "EXEC_STANDARD": "A106",
      "MATERIAL_GRADE_CODE": "GR.B",
      "SPECIAL_REQ": []
    },
    {
      "EXEC_STANDARD": "A53",
      "MATERIAL_GRADE_CODE": "GR.B",
      "SPECIAL_REQ": []
    }
  ]
}
```

### 4.2 `EXEC_STANDARD`

只提取真正的材质标准主体。

常见值：

- `ASTM A403`
- `ASTM A182`
- `ASTM A516`
- `ASTM A234`
- `A182`
- `A53`
- `A106`
- `API5L`

规则：

- 标准在前时直接提取
- 标准在后时也要提取
- 仅提取标准主体，不把牌号、等级、class 混进去

正确示例：

- `ASTM A403 WP316L` -> `EXEC_STANDARD = ASTM A403`
- `F304 A182` -> `EXEC_STANDARD = A182`
- `F304 ASTM A182` -> `EXEC_STANDARD = ASTM A182`
- `WP11 ASTM A234` -> `EXEC_STANDARD = ASTM A234`
- `API5L L245` -> `EXEC_STANDARD = API5L`

### 4.3 `MATERIAL_GRADE_CODE`

`MATERIAL_GRADE_CODE` 存放完整材质牌号表达，按原文表面形式保留。

不要再单独拆分 `LEVEL` 字段。

这意味着以下内容整体保留在 `MATERIAL_GRADE_CODE`：

- `WP316L`
- `WP316L-S`
- `GR.B`
- `GRADE B`
- `Gr.F304`
- `Grade 70`
- `Gr.6`
- `316L_Gr.II`
- `S31603I`
- `X2CrNiMo17-12-2(S31603)`

规则：

- 原文有 `_` 就保留 `_`
- 原文有 `Gr.` 就保留 `Gr.`
- 原文有 `GRADE` 就保留 `GRADE`
- 原文有括号别名且本身构成一个整体表达，可以整体保留
- 依附型等级表达并入 `MATERIAL_GRADE_CODE`，不再拆独立标签

正确示例：

- `316L_Gr.II` -> `MATERIAL_GRADE_CODE = 316L_Gr.II`
- `S31603 GR.III` -> `MATERIAL_GRADE_CODE = S31603 GR.III`
- `ASTM A516 Grade 70` -> `MATERIAL_GRADE_CODE = Grade 70`
- `A333GR.6` -> `MATERIAL_GRADE_CODE = GR.6`
- `SA182 Gr.F304` -> `MATERIAL_GRADE_CODE = Gr.F304`
- `ASTM A182 Grade F 304 (UNS S30400)` -> `MATERIAL_GRADE_CODE = Grade F 304 (UNS S30400)`

### 4.4 `SPECIAL_REQ`

`SPECIAL_REQ` 按原文表面形式提取，不提前归一。

常见值：

- `NACE`
- `CE`
- `ANTI-H2S`
- `GALVANIZED`
- `GALV.`
- `Glav.`
- `Zn`
- `Zn#`

规则：

- 保留原文写法
- 不在一阶段改写成业务归一标签
- 一个材质可以有多个特殊要求

正确示例：

- `20+Zn#` -> `MATERIAL_GRADE_CODE = 20`，`SPECIAL_REQ = ["Zn#"]`
- `20 GALV.` -> `MATERIAL_GRADE_CODE = 20`，`SPECIAL_REQ = ["GALV."]`
- `A53 GR.B GALVANIZED` -> `MATERIAL_GRADE_CODE = GR.B`，`SPECIAL_REQ = ["GALVANIZED"]`
- `20/Glav.` -> `MATERIAL_GRADE_CODE = 20`，`SPECIAL_REQ = ["Glav."]`

## 5. 不提取或进入复核的情况

以下情况不建议强行自动拆分：

- 表达明显不完整，如 `A516 GR`
- 无法判断是单材质还是复合材质
- 阀门内件、trim、复合 BOM 材质

当前建议：

- 无法稳定拆分的样本进入人工复核
- 不要为了追求结构完整而硬补字段

## 6. class 信息处理

以下信息当前不作为材质核心字段提取：

- `CL32`
- `Class 1`
- `Cl.1`

原因：

- 在当前编码体系里这类 class 信息通常不决定材质编码结果
- 因此不单独放入 `EXEC_STANDARD`、`MATERIAL_GRADE_CODE`、`SPECIAL_REQ`

## 8. 一句话规则

材质标注的核心原则是：

**标准单独抽，牌号整体保留，特殊要求原样保留，不做一阶段归一。**
