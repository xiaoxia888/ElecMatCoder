# 一阶段结构化标注草案-其他管件

## 1. 目的

本文件只定义一件事：

- 其他管件模型的一阶段结构化标注方式

这里的前提已经固定：

- 不同分类使用不同模型
- 一个模型一套结构

因此本文件只讨论：

- 其他管件模型应该输出什么结构
- 其他管件模型下 `BODY` 如何归一
- 其他管件模型下 `TYPE` 哪些字段需要标注

---

## 2. 基本原则

### 2.1 一个模型一套结构

既然已经决定按分类拆模型，那么其他管件模型只保留其他管件真正需要的字段。

### 2.2 `BODY` 按编码种类归一

`BODY` 按**最终编码种类**归一。

原则：

- 编码相同，可以归一
- 编码不同，必须分开

当前其他管件模型下，至少需要区分：

- `管箍`
- `单口管箍`
- `双口管箍`
- `异径双口管箍`
- `活接头`
- `管塞`
- `六角头管塞`
- `短节`
- `翻边短节`
- `补强板开口焊`
- `不补强板开口焊`
- `加强管嘴`
- `快速接头`
- `对焊环`

### 2.3 `TYPE` 字段标注分工

其他管件模型下：

- `BODY`：标注其他管件类编码种类
- `MANU`：标注制造方式
- `CONN`：标注连接方式
- `ENDS`：标注端部形式

例如：

- `COUPLING`
  - 应标成 `BODY=管箍`

- `单承口管箍`
  - 应标成 `BODY=单口管箍`

- `双头管箍`
  - 应标成 `BODY=双口管箍`

- `同心异径双口管箍`
  - 应标成 `BODY=异径双口管箍`

- `UNION`
  - 应标成 `BODY=活接头`

- `PLUG`
  - 应标成 `BODY=管塞`

- `HEX HEAD PLUG`
  - 应标成 `BODY=六角头管塞`

- `NIPPLE`
  - 应标成 `BODY=短节`

- `QUICK RELEASE COUPLING`
  - 应标成 `BODY=快速接头`

---

## 3. 其他管件模型的输出结构

当前建议其他管件模型的一阶段输出中，`TYPE` 保留：

```json
"TYPE": {
  "BODY": "",
  "MANU": [],
  "CONN": [],
  "ENDS": []
}
```

说明：

- `BODY`：其他管件类编码种类
- `MANU`：制造方式
- `CONN`：连接方式
- `ENDS`：端部形式

## 5. 其他管件模型下其他 `TYPE` 字段

### 5.1 `MANU`

以下制造方式进入 `MANU`：

- `SMLS`
- `WELDED`
- `ERW`
- `EFW`
- `SAWL`
- `SAWH`
- `FORGED`

补充规则：

- 原文出现 `SEAMLESS` 时，统一标为 `SMLS`
- 原文出现 `无缝` 时，如需要标制造方式，也统一标为 `SMLS`

### 5.2 `CONN`

以下连接方式进入 `CONN`：

- `SW`
- `SF`
- `NPT`
- `THD`

### 5.3 `ENDS`

以下端部形式进入 `ENDS`：

- `SO`
- `FNPT`
- `MNPT`
- `MTE`
- `TSE`
- `FTE`
- `NPTF`

## 7. 当前结论

其他管件模型当前采用以下规则：

- `TYPE` 保留：
  - `BODY`
  - `MANU`
  - `CONN`
  - `ENDS`
- `BODY` 当前至少区分：
  - `管箍`
  - `单口管箍`
  - `双口管箍`
  - `异径双口管箍`
  - `活接头`
  - `管塞`
  - `六角头管塞`
  - `短节`
  - `翻边短节`
  - `补强板开口焊`
  - `不补强板开口焊`
  - `加强管嘴`
  - `快速接头`
  - `对焊环`
- `COUPLING / 管箍 / 同径管箍`
  - 当前统一归到 `管箍`
- `双头管箍 / 双管箍 / 等径双口管箍`
  - 当前统一归到 `双口管箍`
- `同心异径双口管箍 / 异径双头管箍 / 双螺口异径管箍 / REDUCING COUPLING`
  - 当前统一归到 `异径双口管箍`
- `UNION / 活接头`
  - 当前统一归到 `活接头`
- `PLUG / 管塞`
  - 当前统一归到 `管塞`
- `HEX HEAD PLUG / 六角头管塞`
  - 当前统一归到 `六角头管塞`
- `NIPPLE / PIPE NIPPLE / 螺纹短节`
  - 当前统一归到 `短节`
- `QUICK RELEASE COUPLING`
  - 当前统一归到 `快速接头`
