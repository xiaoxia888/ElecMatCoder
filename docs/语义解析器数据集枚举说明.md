# 语义解析器数据集枚举说明

## 1. 目标结构

训练样本结构：

```json
{
  "input": "原始材料描述",
  "output": {
    "mentions": [],
    "semantics": [],
    "decisions": {}
  }
}
```

- `mentions`：证据层，回答“原文里抓到了什么片段”
- `semantics`：语义层，回答“这个片段在当前上下文里代表什么”
- `decisions`：决策层，回答“最终落到哪个字段”

## 2. `mentions.type` 枚举

### `TYPE_TERM`
- 含义：种类相关术语
- 对应来源：`TYPE.BODY`、`TYPE.MANU`、`TYPE.CONN`、`TYPE.ENDS`、`TYPE.SEAL`
- 示例：`ELBOW`、`LAP JOINT FLANGE`、`SMLS`、`RF`

### `SIZE_TERM`
- 含义：尺寸相关术语
- 对应来源：`SIZE.DN`、`SIZE.OD`、`SIZE.INCH`、`SIZE.LENGTH`
- 示例：`DN100`、`89`、`1-1/2"`、`L=300MM`

### `PRESSURE_TERM`
- 含义：压力等级术语
- 对应来源：`PRESSURE`
- 示例：`CL150`、`150lbs`、`PN16`

### `THICKNESS_TERM`
- 含义：壁厚相关术语
- 对应来源：`THICKNESS.MM`、`THICKNESS.INCH`、`THICKNESS.SCHEDULE`、`THICKNESS.SERIES`、`THICKNESS.BWG`
- 示例：`4.5`、`0.065`、`SCH40`、`XS`、`18 BWG`

### `MATERIAL_TERM`
- 含义：材质相关术语
- 对应来源：`MATERIAL.ITEMS[].EXEC_STANDARD`、`MATERIAL.ITEMS[].GRADE`
- 示例：`ASTM A403`、`WP304L`、`A.240`、`Gr 304L`

### `SPECIAL_REQ_TERM`
- 含义：材质特殊要求术语
- 对应来源：`MATERIAL.ITEMS[].SPECIAL_REQ`
- 示例：`NACE`、`GALV`、`Zn`、`ANTI-H2S`

### `STANDARD_TERM`
- 含义：规范相关术语
- 对应来源：`STANDARD[].BODY`、`STANDARD[].GRADE`、`STANDARD[].APPENDIX`、`STANDARD[].METHOD`
- 示例：`ASME B16.5`、`Series I`、`附录A`、`Method E`

## 3. `semantics.semantic_tag` 枚举

### TYPE 类

#### `TYPE_BODY`
- 含义：种类主体片段
- 最终落点：`TYPE.BODY`

#### `TYPE_MANU`
- 含义：制造工艺片段
- 最终落点：`TYPE.MANU`

#### `TYPE_CONN`
- 含义：连接方式片段
- 最终落点：`TYPE.CONN`

#### `TYPE_ENDS`
- 含义：端部形式片段
- 最终落点：`TYPE.ENDS`

#### `TYPE_SEAL`
- 含义：密封面片段
- 最终落点：`TYPE.SEAL`

### SIZE 类

#### `SIZE_DN`
- 含义：公称直径片段
- 最终落点：`SIZE.DN`

#### `SIZE_DN_PAIR`
- 含义：复合公称直径片段
- 说明：一个原文连续片段中同时包含两个 DN 值，例如 `DN150XDN80`
- 最终落点：解析后落到 `SIZE.DN`

#### `SIZE_OD`
- 含义：外径片段
- 最终落点：`SIZE.OD`

#### `SIZE_OD_PAIR`
- 含义：复合外径片段
- 说明：一个原文连续片段中同时包含两个外径值，例如 `φ60.3Xφ48.3`
- 最终落点：解析后落到 `SIZE.OD`

#### `SIZE_INCH`
- 含义：英寸尺寸片段
- 最终落点：`SIZE.INCH`

#### `SIZE_INCH_PAIR`
- 含义：复合英寸尺寸片段
- 说明：一个原文连续片段中同时包含两个英寸值，例如 `6"x2"`
- 最终落点：解析后落到 `SIZE.INCH`

#### `SIZE_LENGTH`
- 含义：长度片段
- 最终落点：`SIZE.LENGTH`

### PRESSURE 类

#### `PRESSURE_CLASS`
- 含义：压力等级片段
- 最终落点：`PRESSURE`

### THICKNESS 类

#### `THICKNESS_MM`
- 含义：毫米壁厚片段
- 最终落点：`THICKNESS.MM`

#### `THICKNESS_MM_PAIR`
- 含义：复合毫米壁厚片段
- 说明：一个原文连续片段中同时包含两个毫米壁厚值，例如 `T=4x3.5mm`、`4mmX3.5mm`
- 最终落点：解析后落到 `THICKNESS.MM`

#### `THICKNESS_INCH`
- 含义：英制壁厚片段
- 最终落点：`THICKNESS.INCH`

#### `THICKNESS_SCHEDULE`
- 含义：Schedule 壁厚片段
- 最终落点：`THICKNESS.SCHEDULE`

#### `THICKNESS_SCHEDULE_PAIR`
- 含义：复合 Schedule 壁厚片段
- 说明：一个原文连续片段中同时包含两个 Schedule 值，例如 `SCH40SXSCH40S`、`S-10S×SCH40S`
- 最终落点：解析后落到 `THICKNESS.SCHEDULE`

#### `THICKNESS_SERIES`
- 含义：Series 壁厚片段
- 最终落点：`THICKNESS.SERIES`

#### `THICKNESS_BWG`
- 含义：BWG 壁厚片段
- 最终落点：`THICKNESS.BWG`

### MATERIAL 类

#### `MATERIAL_EXEC_STANDARD`
- 含义：材质执行标准片段
- 最终落点：`MATERIAL.ITEMS[].EXEC_STANDARD`

#### `MATERIAL_GRADE`
- 含义：材质牌号/等级片段
- 最终落点：`MATERIAL.ITEMS[].GRADE`

#### `MATERIAL_SPECIAL_REQ`
- 含义：材质特殊要求片段
- 最终落点：`MATERIAL.ITEMS[].SPECIAL_REQ`

### STANDARD 类

#### `STANDARD_BODY`
- 含义：规范主体片段
- 最终落点：`STANDARD[].BODY`

#### `STANDARD_COMPOSITE`
- 含义：复合规范片段，一个原文证据中同时包含 `STANDARD` 的多个子字段，如 `BODY/GRADE/METHOD/APPENDIX`
- 最终落点：解析后分别进入对应的 `STANDARD[]` 子字段

#### `STANDARD_GRADE`
- 含义：规范等级/系列片段
- 最终落点：`STANDARD[].GRADE`

#### `STANDARD_APPENDIX`
- 含义：规范附录片段
- 最终落点：`STANDARD[].APPENDIX`

#### `STANDARD_METHOD`
- 含义：规范方法片段
- 最终落点：`STANDARD[].METHOD`

## 4. 说明

### `mentions.type` 和 `semantics.semantic_tag` 的区别

- `mentions.type` 是粗分类，表示这段文本大致属于哪一类术语
- `semantics.semantic_tag` 是细分类，表示这段文本在当前上下文里的最终语义角色

对于复合表达，推荐规则是：

- `mentions` 保留原文整体证据片段
- `decisions` 保留拆分后的结构化结果

例如：

```json
{
  "text": "DN150XDN80",
  "type": "SIZE_TERM"
}
```

对应：

```json
{
  "mention_id": "m1",
  "semantic_tag": "SIZE_DN_PAIR"
}
```

最终：

```json
"SIZE": {
  "DN": ["DN150", "DN80"]
}
```

壁厚同理：

```json
{
  "text": "SCH40SXSCH40S",
  "type": "THICKNESS_TERM"
}
```

对应：

```json
{
  "mention_id": "m2",
  "semantic_tag": "THICKNESS_SCHEDULE_PAIR"
}
```

其他复合片段同理：

- `φ60.3Xφ48.3` -> `SIZE_OD_PAIR`
- `6"x2"` -> `SIZE_INCH_PAIR`
- `T=4x3.5mm` -> `THICKNESS_MM_PAIR`

例如：

```json
{
  "id": "m1",
  "text": "A.240",
  "type": "MATERIAL_TERM"
}
```

对应：

```json
{
  "mention_id": "m1",
  "semantic_tag": "MATERIAL_EXEC_STANDARD"
}
```

### 哪些字段不生成 mention

以下字段默认只保留在 `decisions`，不强行生成 mention：

- `MATERIAL.RELATION`

原因：
- 它描述的是多个材质条目之间的关系
- 不是原文中的独立证据片段

## 5. 自动转换脚本

脚本：

- [scripts/convert_stage1_to_semantic_parser_dataset.py](/Users/guoxi/Desktop/workspace/NJNCC/python_code/ElecMatCoder/scripts/convert_stage1_to_semantic_parser_dataset.py)

用途：

- 将当前一阶段数据集自动转换为 `mentions + semantics + decisions` 结构
- `decisions` 保留原样
- `mentions/semantics` 依据当前 `decisions` 机械推导生成

限制：

- 它不能补出你没有标注过的隐藏语义
- 它也不能保证 `mention.text` 一定是原文中的最佳证据片段
- 它更适合先生成第一版语义解析训练集，再人工抽样修正
