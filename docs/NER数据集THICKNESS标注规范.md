# NER 数据集 THICKNESS 标注规范

## 1. 提取什么

从原文中提取所有与壁厚有关、能用于后续编码的壁厚信息，按类别拆到 `THICKNESS` 对象中：

- `MM`：毫米数值壁厚
- `INCH`：英寸数值壁厚
- `SCHEDULE`：表号壁厚
- `SERIES`：重量系列壁厚
- `BWG`：线规壁厚

格式：

```json
"THICKNESS": {
  "MM": [],
  "INCH": [],
  "SCHEDULE": [],
  "SERIES": [],
  "BWG": []
}
```

空数组不输出。

## 2. 值格式怎么统一

1. `MM` 统一只保留数字本体，不保留 `mm`
2. `INCH` 统一只保留数字本体，不保留 `in` 或 `"`
3. `SCHEDULE` 统一保留 `SCH` 前缀
4. `SERIES` 统一保留系列名本体
5. `BWG` 统一只保留规号数字，不保留 `BWG`

## 3. 归类规则

### `MM`

以下都归 `MM`：

- `3.5`
- `4.6mm`
- `4.0 mm`
- `T=4x3.5mm` -> `MM = ["4", "3.5"]`
- `10mm X8mm` -> `MM = ["10", "8"]`

### `INCH`

以下都归 `INCH`：

- `0.049 in`
- `0.065 in`
- `THK:0.035`

前提：当前样本是英制管/管子上下文，能稳定判断该数值是英寸壁厚。

### `SCHEDULE`

以下都归 `SCHEDULE`：

- `SCH40`
- `SCH10S`
- `SCH160`
- `SCH 40`
- `S-40`
- `40S`

统一格式：

- `SCH 40` -> `SCH40`
- `S-40` -> `SCH40`
- `40S` -> `SCH40S`

### `SERIES`

以下都归 `SERIES`：

- `STD`
- `XS`
- `XXS`
- `S-STD` -> `STD`
- `S-XS` -> `XS`
- `Ⅱ-3`

如果原文出现多个系列值，则按原文顺序保留：

- `Ⅱ-3XⅡ-2.5` -> `SERIES = ["Ⅱ-3", "Ⅱ-2.5"]`

### `BWG`

以下都归 `BWG`：

- `18 BWG`
- `16 BWG`
- `14 BWG`

统一格式：

- `18 BWG` -> `18`

## 4. 多值怎么存

1. 同一类别出现多个值时，按原文顺序放入数组
2. 重复值去重后保留一次
3. 不同类别可以同时保留

## 5. `(L)` / `(S)` 规则

`(L)` / `(S)` 保留：

- `SCH40(L)` 保留
- `XS(S)` 保留
- `3.5(L)` 保留

## 6. 注意事项

1. `THICKNESS` 追求的是编码友好的统一输出，不是原样抄写
2. 分类信息放在 key 里，值本身不重复带单位
3. `STD / XS / XXS` 不强行转换成 `SCH`
4. 无法稳定判断时，进入人工复核，不强行猜测
