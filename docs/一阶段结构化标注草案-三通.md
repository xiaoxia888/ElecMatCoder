# 一阶段结构化标注草案-三通

## 1. 目的

本文件只定义一件事：

- 三通模型的一阶段结构化标注方式

这里的前提已经固定：

- 不同分类使用不同模型
- 一个模型一套结构

因此本文件只讨论：

- 三通模型应该输出什么结构
- 三通模型下 `BODY` 如何归一
- 三通模型下 `TYPE` 哪些字段需要标注

---

## 2. 基本原则

### 2.1 一个模型一套结构

既然已经决定按分类拆模型，那么三通模型只保留三通真正需要的字段。

### 2.2 `BODY` 按编码种类归一

`BODY` 按**最终编码种类**归一。

原则：

- 编码相同，可以归一
- 编码不同，必须分开

当前三通模型下，至少需要区分：

- `三通`
- `等径三通`
- `异径三通`
- `斜三通`
- `异径斜三通`

### 2.3 `TYPE` 字段标注分工

三通模型下：

- `BODY`：标注三通类编码种类
- `GEOMETRY.ANGLE`：标注斜三通角度
- `MANU`：标注制造方式
- `CONN`：标注连接方式
- `ENDS`：标注端部形式

例如：

- `TEE STRAIGHT`
  - 应标成 `BODY=等径三通`

- `REDUCING TEE`
  - 应标成 `BODY=异径三通`

- `45Deg Lateral`
  - 应标成 `BODY=斜三通`, `GEOMETRY.ANGLE=45`

- `90Deg Lateral`
  - 应标成 `BODY=斜三通`, `GEOMETRY.ANGLE=90`

- `螺纹三通`
  - 应标成 `BODY=三通 / 等径三通 / 异径三通`, `CONN=[THD]`

- `承插焊三通`
  - 应标成 `BODY=三通 / 等径三通 / 异径三通`, `CONN=[SW]`

---

## 3. 三通模型的输出结构

当前建议三通模型的一阶段输出中，`TYPE` 保留：

```json
"TYPE": {
  "BODY": "",
  "GEOMETRY": {
    "ANGLE": ""
  },
  "MANU": [],
  "CONN": [],
  "ENDS": []
}
```

说明：

- `BODY`：三通类编码种类
- `GEOMETRY.ANGLE`：斜三通角度
- `MANU`：制造方式
- `CONN`：连接方式
- `ENDS`：端部形式

---

## 4. 三通模型下 `BODY` 当前建议

当前至少保留以下标准 `BODY`：

- `三通`
- `等径三通`
- `异径三通`
- `斜三通`
- `异径斜三通`

### 4.1 `三通`

适用于：

- `三通`
- `TEE`
- `Tee`
- 其他原文只明确为普通三通，且未明确说明等径/异径/斜三通的表达

### 4.2 `等径三通`

适用于：

- `等径三通`
- `TEE STRAIGHT`
- `STRAIGHT TEE`
- `Equal Tee`
- `TEE STRAIGHT`
- 其他最终编码为等径三通的表达

### 4.3 `异径三通`

适用于：

- `异径三通`
- `TEE REDUCED`
- `REDUCING TEE`
- `RED TEE`
- `Reducing Tee`
- `Red. Tee`
- `RED.TEE`
- 其他最终编码为异径三通的表达

### 4.4 `斜三通`

适用于：

- `斜三通`
- `Lateral`
- `Lateral Tee`
- `45Deg Lateral`
- `90Deg Lateral`
- `45度 Lateral`
- `90度 Lateral`
- 其他最终编码为斜三通的表达

### 4.5 `异径斜三通`

适用于：

- `异径斜三通`
- `45度异径斜三通`
- 其他最终编码为异径斜三通的表达

### 4.6 `BODY` 完整归一化映射

以下为当前数据集中已出现写法到标准 `BODY` 的归一化结果。

#### 归一到 `三通`
- `TEE` -> `三通`
- `三通` -> `三通`

#### 归一到 `等径三通`
- `TEE STRAIGHT` -> `等径三通`
- `STRAIGHT TEE` -> `等径三通`
- `Equal Tee` -> `等径三通`

#### 归一到 `异径三通`
- `REDUCING TEE` -> `异径三通`
- `TEE REDUCED` -> `异径三通`
- `RED TEE` -> `异径三通`

#### 归一到 `斜三通`
- `Lateral` -> `斜三通`
- `Lateral Tee` -> `斜三通`
- `45Deg Lateral` -> `斜三通`
- `90Deg Lateral` -> `斜三通`

#### 归一到 `异径斜三通`
- `异径斜三通` -> `异径斜三通`

---

## 5. 三通模型下其他 `TYPE` 字段

### 5.1 `GEOMETRY.ANGLE`

以下角度进入 `ANGLE`：

- `45`
- `90`

原文出现斜三通角度时，直接提取数字。

### 5.2 `MANU`

以下制造方式进入 `MANU`：

- `SMLS`
- `WELDED`
- `ERW`
- `EFW`
- `SAWL`
- `SAWH`
- `FORGED`

补充规则：

- 原文出现 `SEAMLESS` 时，统一标为 `SMLS`
- 原文出现 `无缝` 时，如需要标制造方式，也统一标为 `SMLS`

### 5.3 `CONN`

以下连接方式进入 `CONN`：

- `SW`
- `SF`
- `NPT`
- `THD`

### 5.4 `ENDS`

以下端部形式进入 `ENDS`：

- `FNPT`
- `MNPT`
- `MTE`
- `TSE`
- `FTE`
- `NPTF`

## 7. 当前结论

三通模型当前采用以下规则：

- `TYPE` 保留：
  - `BODY`
  - `GEOMETRY`
  - `MANU`
  - `CONN`
  - `ENDS`
- `BODY` 当前至少区分：
  - `三通`
  - `等径三通`
  - `异径三通`
  - `斜三通`
  - `异径斜三通`
- `GEOMETRY.ANGLE` 用于承接斜三通角度
- `螺纹`
  - 通过 `CONN=[NPT]` 或 `CONN=[THD]` 承接
- `承插焊`
  - 通过 `CONN=[SW]` 承接
