# 一阶段结构化标注草案-变径件

## 1. 目的

本文件只定义一件事：

- 变径件模型的一阶段结构化标注方式

这里的前提已经固定：

- 不同分类使用不同模型
- 一个模型一套结构

因此本文件只讨论：

- 变径件模型应该输出什么结构
- 变径件模型下 `BODY` 如何归一
- 变径件模型下 `TYPE` 哪些字段需要标注

---

## 2. 基本原则

### 2.1 一个模型一套结构

既然已经决定按分类拆模型，那么变径件模型只保留变径件真正需要的字段。

### 2.2 `BODY` 按编码种类归一

`BODY` 按**最终编码种类**归一。

原则：

- 编码相同，可以归一
- 编码不同，必须分开

当前变径件模型下，至少需要区分：

- `异径管`
- `同心异径管`
- `偏心异径管`
- `异径短节`
- `同心异径短节`
- `偏心异径短节`

### 2.3 `TYPE` 字段标注分工

变径件模型下：

- `BODY`：标注变径件类编码种类
- `MANU`：标注制造方式
- `CONN`：标注连接方式
- `ENDS`：标注端部形式

例如：

- `REDUCER`
  - 应标成 `BODY=异径管`

- `CONCENTRIC REDUCER`
  - 应标成 `BODY=同心异径管`

- `ECCENTRIC REDUCER`
  - 应标成 `BODY=偏心异径管`

- `SWAGE NIPPLE`
  - 应标成 `BODY=同心异径短节`

- `ECC SWAGE NIPPLE`
  - 应标成 `BODY=偏心异径短节`

---

## 3. 变径件模型的输出结构

当前建议变径件模型的一阶段输出中，`TYPE` 保留：

```json
"TYPE": {
  "BODY": "",
  "MANU": [],
  "CONN": [],
  "ENDS": []
}
```

说明：

- `BODY`：变径件类编码种类
- `MANU`：制造方式
- `CONN`：连接方式
- `ENDS`：端部形式

---

## 4. 变径件模型下 `BODY` 当前建议

当前至少保留以下标准 `BODY`：

- `异径管`
- `同心异径管`
- `偏心异径管`
- `异径短节`
- `同心异径短节`
- `偏心异径短节`

### 4.1 `异径管`

适用于：

- `异径管`
- `REDUCER`
- 其他原文只明确为异径管，未明确同心/偏心的表达

### 4.2 `同心异径管`

适用于：

- `同心异径管`
- `CONCENTRIC REDUCER`
- `CON. REDUCER`
- `Concentric reducer`
- 其他最终编码为同心异径管的表达

### 4.3 `偏心异径管`

适用于：

- `偏心异径管`
- `ECCENTRIC REDUCER`
- `ECC. REDUCER`
- `Eccentric reducer`
- 其他最终编码为偏心异径管的表达

### 4.4 `异径短节`

适用于：

- `异径短节`
- 其他原文只明确为异径短节，未明确同心/偏心的表达

### 4.5 `同心异径短节`

适用于：

- `同心异径短节`
- `SWAGE NIPPLE`
- `Swage Nipple`
- `CON SWAGE NIPPLE`
- `CONCENTRIC SWAGE`
- 其他最终编码为同心异径短节的表达

### 4.6 `偏心异径短节`

适用于：

- `偏心异径短节`
- `ECC SWAGE NIPPLE`
- `ECC. SWAGE NIPPLE`
- `ECCENTRIC SWAGE NIPPLE`
- 其他最终编码为偏心异径短节的表达

### 4.7 `BODY` 完整归一化映射

以下为当前数据集中已出现写法到标准 `BODY` 的归一化结果。

#### 归一到 `异径管`
- `REDUCER` -> `异径管`
- `异径管` -> `异径管`

#### 归一到 `同心异径管`
- `CONCENTRIC REDUCER` -> `同心异径管`

#### 归一到 `偏心异径管`
- `ECCENTRIC REDUCER` -> `偏心异径管`

#### 归一到 `异径短节`
- `异径短节` -> `异径短节`

#### 归一到 `同心异径短节`
- `SWAGE NIPPLE` -> `同心异径短节`
- `CONCENTRIC SWAGE` -> `同心异径短节`

#### 归一到 `偏心异径短节`
- `ECC SWAGE NIPPLE` -> `偏心异径短节`
- `ECCENTRIC SWAGE NIPPLE` -> `偏心异径短节`

## 5. 变径件模型下其他 `TYPE` 字段

### 5.1 `MANU`

以下制造方式进入 `MANU`：

- `SMLS`
- `WELDED`
- `ERW`
- `EFW`
- `SAWL`
- `SAWH`
- `FORGED`

补充规则：

- 原文出现 `SEAMLESS` 时，统一标为 `SMLS`
- 原文出现 `无缝` 时，如需要标制造方式，也统一标为 `SMLS`

### 5.2 `CONN`

以下连接方式进入 `CONN`：

- `SW`
- `SF`
- `NPT`
- `THD`

### 5.3 `ENDS`

以下端部形式进入 `ENDS`：

- `SO`
- `FNPT`
- `MNPT`
- `MTE`
- `TSE`
- `FTE`
- `NPTF`

## 7. 当前结论

变径件模型当前采用以下规则：

- `TYPE` 保留：
  - `BODY`
  - `MANU`
  - `CONN`
  - `ENDS`
- `BODY` 当前至少区分：
  - `异径管`
  - `同心异径管`
  - `偏心异径管`
  - `异径短节`
  - `同心异径短节`
  - `偏心异径短节`
- `SWAGE NIPPLE`
  - 归到 `同心异径短节`
- `ECC SWAGE NIPPLE`
  - 归到 `偏心异径短节`

后续待补充：

- `异径管` 与 `同心异径管` 的边界例子
- `异径管` 与 `偏心异径管` 的边界例子
- `异径短节` 与 `同心/偏心异径短节` 的边界例子
