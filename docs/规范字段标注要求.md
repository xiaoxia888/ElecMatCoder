# 规范字段标注要求

本文用于约束 `ner_data.json` 中规范相关字段的标注方式，目标是：

- 将规范主体与其附加信息拆开标
- 保留规范之间的绑定关系
- 尽量保持原文语义，不做无必要的翻译或改写

适用字段：

- `STANDARD`
- `STANDARD_GRADE`
- `STANDARD_APPENDIX`
- `STANDARD_METHOD`

## 1. 总体原则

### 1.1 只拆字段，不做跨语言改写

标注时允许将粘连写法拆开，但**不允许把中文改成英文，也不允许把英文改成中文**。

正确示例：

- 原文：`方法B` -> `STANDARD_METHOD = 方法B`
- 原文：`Method B` -> `STANDARD_METHOD = Method B`
- 原文：`SeriesI` -> `STANDARD_GRADE = Series I`
- 原文：`I系列` -> `STANDARD_GRADE = I系列`

错误示例：

- 原文：`方法B` -> `STANDARD_METHOD = Method B`
- 原文：`I系列` -> `STANDARD_GRADE = Series I`

说明：

- `SeriesI -> Series I` 属于边界拆分，不属于翻译，允许。
- `方法B -> Method B` 属于中英改写，不允许。

### 1.1.1 允许的轻量规范化

只允许做不改变语言和词面的轻量规范化：

- 补开粘连边界
- 清理多余空格
- 保留原词面，不做跨语言或跨词面改写

允许示例：

- `SeriesI` -> `Series I`
- `SerialI` -> `Series I`
- `SerI` -> `Series I`
- `MethodE` -> `Method E`
- `方法E` -> `方法E`

不允许示例：

- `方法E` -> `Method E`
- `Method E` -> `方法E`
- `I系列` -> `Series I`

说明：

- `Series / Serial / Ser` 视为同一英文等级表达的不同写法，可统一标为 `Series X`。
- 中文 `I系列 / II系列` 保持中文，不转成英文。
- 可以补空格，也允许把英文简写/误写归一到标准英文写法。

### 1.2 规范主体与修饰信息必须拆开

以下信息不应继续混在 `STANDARD` 中：

- 等级：`Series I`、`Ia`、`II`、`I系列`、`I类填充金属`
- 附录：`附录B`
- 方法：`方法E`、`Method E`

拆分后：

- `STANDARD` 只保留规范主体
- 其他信息分别进入对应字段

### 1.3 附加字段必须绑定到具体规范

当一条样本中存在多个规范时，`STANDARD_GRADE` / `STANDARD_APPENDIX` / `STANDARD_METHOD` 必须说明它们属于哪一个规范。

统一采用对象数组格式：

```json
{
  "STANDARD": ["SH/T3410", "GB/T19326", "NB/T47010"],
  "STANDARD_GRADE": [
    {"value": "Series I", "bind_to_index": 1}
  ],
  "STANDARD_METHOD": [
    {"value": "方法E", "bind_to_index": 2}
  ]
}
```

其中：

- `bind_to_index = 1` 表示绑定到 `STANDARD[1]`
- `bind_to_index = 2` 表示绑定到 `STANDARD[2]`

## 2. 各字段定义

### 2.1 `STANDARD`

定义：规范主体，不包含等级、附录、方法、年份。

标注要求：

- 只保留规范本体
- 不包含 `Series I` / `Ia` / `附录B` / `方法E`
- 不包含年份（如 `2017`、`2016`）

示例：

- 原文：`GB/T19326SeriesI`
  - `STANDARD = GB/T19326`
- 原文：`NB/T47010 方法E`
  - `STANDARD = NB/T47010`
- 原文：`GB/T 12459-2017附录B`
  - `STANDARD = GB/T12459`

### 2.2 `STANDARD_GRADE`

定义：规范等级或等级性修饰信息。

本项目中以下内容归入 `STANDARD_GRADE`：

- `Series I`
- `Series II`
- `Ia`
- `I`
- `II`
- `I系列`
- `II系列`
- `I类填充金属`
- `II类填充金属`

说明：

- `I类填充金属 / II类填充金属` 在本项目中视为可编码等级信息。
- 例如：`GB/T12771 + I类填充金属` 最终编码为 `GBT12771I`。
- `Series I`、`Serial I`、`Ser I` 都属于等级信息，标注时统一为 `Series I`。

补充示例：

- 原文：`GB/T14383SeriesI`
  - `STANDARD = GB/T14383`
  - `STANDARD_GRADE = Series I`
- 原文：`GB/T14383SerialI`
  - `STANDARD = GB/T14383`
  - `STANDARD_GRADE = Series I`
- 原文：`GB/T14383SerI`
  - `STANDARD = GB/T14383`
  - `STANDARD_GRADE = Series I`

示例：

```json
{
  "STANDARD": ["GB/T12771"],
  "STANDARD_GRADE": [
    {"value": "I类填充金属", "bind_to_index": 0}
  ]
}
```

### 2.3 `STANDARD_APPENDIX`

定义：规范附录信息。

当前归入该字段的内容：

- `附录A`
- `附录B`
- `附录C`

示例：

```json
{
  "STANDARD": ["GB/T12459"],
  "STANDARD_APPENDIX": [
    {"value": "附录B", "bind_to_index": 0}
  ]
}
```

### 2.4 `STANDARD_METHOD`

定义：规范方法信息。

注意：

- 原文是中文，就保留中文，如 `方法A`、`方法B`
- 原文是英文，就保留英文，如 `Method A`、`Method B`
- 不做中英互转
- `MethodB` 可拆为 `Method B`
- `方法B` 保持 `方法B`

说明：

- `方法B` 和 `Method B` 都合法
- 但它们不是同一个标注值，不能为了统一格式相互替换

示例：

```json
{
  "STANDARD": ["NB/T47010"],
  "STANDARD_METHOD": [
    {"value": "方法E", "bind_to_index": 0}
  ]
}
```

或

```json
{
  "STANDARD": ["NB/T47010"],
  "STANDARD_METHOD": [
    {"value": "Method E", "bind_to_index": 0}
  ]
}
```

## 3. 绑定规则

### 3.1 `STANDARD` 顺序规则

`STANDARD` 按原文出现顺序排列。

### 3.2 修饰项绑定规则

一个修饰项默认绑定到它语义上所属的规范主体。

示例：

原文：

```text
对焊管接台, SH/T3410-S30408, BE, GB/T 19326 Series I, DN100 x DN20, _NB/T47010 方法E-S30408
```

标注：

```json
{
  "STANDARD": ["SH/T3410", "GB/T19326", "NB/T47010"],
  "STANDARD_GRADE": [
    {"value": "Series I", "bind_to_index": 1}
  ],
  "STANDARD_METHOD": [
    {"value": "方法E", "bind_to_index": 2}
  ]
}
```

### 3.3 无法明确绑定时

若人工也无法确定某个等级/附录/方法属于哪个规范：

- 不要强行标注
- 该样本应进入人工复核清单

## 5. 不允许的标注方式

以下方式不允许：

- 把 `方法B` 统一改成 `Method B`
- 把 `I系列` 统一翻译成 `Series I`
- 把 `附录B` 仍然留在 `STANDARD` 中不拆
- 一个样本里有多个规范，但等级不写 `bind_to_index`
- 为了凑统一格式，把无法确定的修饰信息强行挂到某个规范上

## 6. 标注建议

执行顺序建议如下：

1. 先识别原文中的所有规范主体
2. 再识别等级、附录、方法
3. 判断每个修饰项属于哪个规范
4. 写入 `bind_to_index`
5. 若不能确定绑定关系，进入人工复核

## 7. 备注

本文是标注规范，不是编码规范。

也就是说：

- 标注层要尽量忠实于原文
- 编码层如果后续需要做归一化，可以在编码阶段再处理
- 不能在标注阶段提前做中英翻译或过度归一化
